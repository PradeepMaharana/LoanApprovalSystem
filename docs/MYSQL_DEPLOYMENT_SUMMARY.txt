📊 DATABASE STRUCTURE
════════════════════════════════════════════════════════════════════════════════

Database Name: loan_approval_system

┌─ Table: applicants
│  └─ 1000 records with applicant profile information
│  └─ Columns: applicant_id, age, income, employment_type, location, timestamps
│  └─ Indexes: applicant_id (unique), employment_type, location
│
├─ Table: loan_applications
│  └─ 1000 records with loan details and status
│  └─ Columns: applicant_id, credit_score, loan_amount, tenure_months,
│     liabilities, application_status, risk_score, risk_level, timestamps
│  └─ Indexes: application_status, risk_level, credit_score
│  └─ Foreign Key: applicant_id → applicants
│
├─ Table: risk_assessments
│  └─ 1000 records with detailed risk analysis
│  └─ Columns: applicant_id, credit_score_impact, dti_ratio, dti_impact,
│     age_impact, lti_ratio, lti_impact, final_score
│  └─ Foreign Key: applicant_id → applicants
│
└─ Table: chat_messages
   └─ For storing chat interactions (empty initially)
   └─ Columns: applicant_id, user_message, bot_response, timestamps
   └─ Foreign Key: applicant_id → applicants

1. INITIALIZATION
   ├─ Connects to MySQL server

   ├─ Creates applicants table

   ├─ Creates risk_assessments table

   └─ Sets up foreign key relationships

4. DATA LOADING
   ├─ Reads 1000 records from CSV file
   ├─ Inserts applicant information
   ├─ Calculates risk scores using 4-factor model:
   │  ├─ Credit score factor (±40 points)

   ├─ Determines application status:
   │  ├─ Risk ≥ 75: APPROVED
   │  ├─ Risk 20-74: UNDER_REVIEW

⚠️ TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

Problem: MySQL service not running
Solution:
├─ sudo systemctl start mysql
├─ Check status: sudo systemctl status mysql
└─ Enable auto-start: sudo systemctl enable mysql

Problem: Access denied for user 'root'
Solution:
├─ Try: mysql -u root (without -p)
├─ Or reset password: sudo mysql -u root
└─ See MYSQL_SETUP_GUIDE.md for details

Problem: CSV file not found
Solution:
├─ Verify path: ls Loan_Applicants_Sample_Data.csv
├─ Update path in mysql_setup.py if needed
└─ Ensure sample data files are generated first

Problem: Connection refused on port 3306
Solution:
├─ Check MySQL listening: sudo netstat -tlnp | grep 3306
├─ Verify firewall: sudo ufw allow 3306
└─ Restart MySQL: sudo systemctl restart mysql

Problem: Duplicate entry for key
Solution:
├─ Database exists from previous run
├─ Script automatically drops it (expected)
├─ Wait for script to complete
└─ Check for application_id uniqueness
